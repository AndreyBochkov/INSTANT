What is it?

This is a messenger focused on safety and speed. The lack of trackers, data collectors and modern encryption algorithms rovide these qualities. The project was developed and launched by one person within the framework of individual work.

What are the objectives of INSTANT?

It is immediately necessary to explain that this is a non-commercial project. INSTANT does not aim to provide you with personalized advertising, does not aim to raise money. INSTANT is here for the secure exchange of bytes between two devices connected to the Web.

Security without HTTPS? I can see, "http://i..."

Indeed, INSTANT does not use HTTPS on the site. However, the application uses a special encryption protocol based on the simplified Diffie-Hellman algorithm of TLS 1.3, and so that it works incredibly fast. In simple terms, TLS does a lot more for compatibility with any device, and the INSTANT protocol only works between your phone and that server, so it can set a connection for a smaller number of packets.

Everyone collects data on everyone, it is obvious.

But not private messengers. Yes, this is only for Android, but for iOS there is already a "bitchat" that combines users into a decentralized network (without servers), guaranteeing anonymity. INSTANT ensures performance - thanks to centralization. How many startups, so many approaches. But it is profitable to collect data for advertisers, not programmers.

Technical limitations?

Alas, there is. INSTANT did not negotiate with smartphone manufacturers about the absolute resolution of background activity. INSTANT does not use Google Firebase Cloud Messaging technology to avoid sharing any data with third-party servers. Because of this, when the screen is turned off, messages are not delivered. So far, INSTANT is more about safety than about speed, but work is underway to improve.
